# 同步登录 + 详细统计 设计文档

**日期：** 2026-05-31
**项目：** myhostex-assistant
**版本：** v3.13.x

---

## 1. 需求概述

为 myhostex-assistant 添加云端同步的登录认证功能，与 csBaby 客户端保持一致：
1. 同步前必须先登录（邮箱 + 密码）
2. 同步后显示详细的同步内容详情

---

## 2. 架构设计

### 2.1 组件结构

```
popup.html (UI)
├── 同步状态区（已登录UI）
├── 登录对话框 (LoginDialog)
└── 注册对话框 (RegisterDialog)

popup-sync.js
├── SyncAuthManager - 认证状态管理
├── SyncUI - UI 控制器
└── SyncService (已存在) - 数据导出/导入

云端 API (csBaby-server)
├── POST /auth/login
├── POST /auth/register
└── POST /sync/upload
```

### 2.2 认证状态存储

```javascript
// chrome.storage.local
{
  sync_auth: {
    userId: "xxx",
    tenantId: "xxx",
    accessToken: "xxx",
    refreshToken: "xxx",
    expiresAt: timestamp
  }
}
```

## 4. 同步数据类型

与 csBaby 保持一致，共 7 类数据：

| 类型 | 说明 |
|------|------|
| 知识库 (replyRules) | 关键词回复规则 |
| AI模型 (aiConfigs) | AI 模型配置 |
| 监控应用 (rooms) | 房源/房间信息 |
| 场景 (scenarios) | 场景配置 |
| 回复历史 (replyHistory) | 回复历史记录 |
| 黑名单 (blacklist) | 消息黑名单 |
| 风格画像 (userStyle) | 用户风格配置 |

---

## 5. 同步详情显示格式

### 5.1 无变更时不显示

- 没有变更的数据类型完全隐藏，不占用 UI 空间
- 有变更的类型才显示

### 5.2 显示格式

```
✓ 同步完成（2024-01-15 14:30）

📤 推送变更：
   • 知识库 +2 ↑1
   • AI模型 +0 ↑1
   • 监控应用 +1

📥 拉取变更：
   • 知识库 5 条
   • 回复历史 2 条
```

**格式说明：**
- `+N` 表示新增数量
- `↑M` 表示更新数量（仅当有更新时显示）
- 纯数字表示拉取的数据条数

## 6. API 接口

### 6.1 登录

```
POST /auth/login
Request: { email: string, password: string }
Response: {
  isSuccess: boolean,
  message: string,
  data: {
    userId: string,
    tenantId: string,
    accessToken: string,
    refreshToken: string,
    expiresAt: number
  }
}
```

### 6.2 注册

```
POST /auth/register
Request: { email: string, password: string, displayName: string }
Response: 同登录接口
```

### 6.3 上传同步数据

```
POST /sync/upload
Headers: Authorization: Bearer <token>
Request: {
  data: string (JSON),
  timestamp: string,
  deviceId: string,
  stats: {
    inserted: number,
    updated: number,
    deleted: number,
    byType: { [type: string]: { inserted, updated, deleted } }
  }
}
```

## 7. 错误处理

| 错误场景 | 用户提示 |
|---------|---------|
| 网络错误 | "网络错误，请检查网络连接" |
| 邮箱或密码错误 | "邮箱或密码错误" |
| Token 过期 | 自动刷新，失败则提示"登录已过期，请重新登录" |
| 同步失败 | 显示具体错误信息 + "重试" 按钮 |

## 10. 验收标准

1. ✅ 未登录时点击"云端同步"，弹出登录对话框
2. ✅ 可以注册新账号
3. ✅ 可以登录已有账号
4. ✅ 登录状态持久化，刷新页面后保持登录
5. ✅ 云端同步后显示详细的推送/拉取变更（只显示有变更的类型）
6. ✅ 登出后清除登录状态
7. ✅ 本地导出/导入无需登录

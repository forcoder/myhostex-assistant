# 更新日志 v3.9.0 - 性能优化：知识库规则优先

## 更新时间
2026-04-06

## 更新内容

### 性能优化
- **知识库规则优先**：当知识库规则匹配成功时，直接返回规则回复，跳过 AI 生成
  - 大幅提升响应速度（无需等待 API 调用）
  - 降低 API 调用成本
  - 确保规则回复的准确性和一致性

### 修改文件
1. **background.js**
   - 在 `handleGenerateSuggestions` 函数中添加逻辑分支
   - 如果 `kbDirectSuggestions.length > 0`：直接返回规则回复，跳过 AI 生成
   - 否则：调用 AI 生成建议

2. **manifest.json**
   - 版本号更新至 3.9.0
   - 更新描述说明

## 技术说明

### 优化前的流程
```
1. 知识库匹配 → 匹配到规则
2. 构建 System Prompt（包含规则内容）
3. 调用 LLM API（等待 1-5 秒）
4. 解析 AI 回复
5. 合并知识库建议和 AI 建议
6. 返回结果
```
**总耗时：** 1-5 秒（取决于 API 响应时间）

### 优化后的流程
```
1. 知识库匹配 → 匹配到规则
2. 直接返回规则回复
```
**总耗时：** < 100ms（几乎即时）

### 代码逻辑

```javascript
const kbDirectSuggestions = matchedEntries
  .filter((e) => e.reply_content && e.reply_content.trim().length > 0)
  .map((e) => e.reply_content.trim());

// ★ 知识库匹配成功，直接返回，跳过 AI 生成
if (kbDirectSuggestions.length > 0) {
  const suggestions = kbDirectSuggestions.slice(0, maxSuggestions);
  console.log("[MyHostex助手][BG] ✅ 知识库匹配成功，直接返回规则建议，跳过 AI 生成");
  return {
    suggestions,
    fromKB: true,
    kbCount: kbDirectSuggestions.length,
    usedModel: "知识库规则",
  };
}

// ★ 知识库匹配失败，使用 AI 生成建议
console.log("[MyHostex助手][BG] 知识库无匹配，使用 AI 生成建议");
// ... 调用 AI API
```

## 使用说明

### 场景 1：常见问题（有规则）
客户发送："价格多少？"
1. 知识库匹配成功（关键词：价格）
2. 直接返回规则回复："您好，平台上能选、能下单的日期，都有房。您看到的价格就是实付价，已叠加所有优惠，是最优惠的。"
3. **耗时：< 100ms**

### 场景 2：复杂问题（无规则）
客户发送："如果临时想提前一天入住可以吗？另外想知道泳池是恒温的吗？"
1. 知识库匹配失败
2. 调用 LLM API 生成建议
3. 返回 AI 生成的回复
4. **耗时：1-5 秒**

# MyHostex 智能回复助手 v3.10.0 - 调试提示增强

## 更新时间
2026-04-06

## 问题修复

### 问题描述
用户反馈："没有匹配到回复规则，为啥没有生成AI建议回复，而且在'等待新消息'"

### 问题分析
经过代码分析，发现了几个可能导致建议未生成的原因：

1. **DOM 消息方向判断错误**：DOM 中的消息被错误标记为 `isOutgoing: true`（房东发送），导致跳过生成
2. **DOM 渲染延迟**：2000ms 延迟后 DOM 还没渲染完成，无法获取消息
3. **没有配置 AI 模型**：虽然会使用本地回退，但用户可能不知道
4. **所有模型都失败**：API 调用失败，但可能被静默捕获

### 修复方案

#### 1. 增加用户提示（content.js）

**位置 1**: DOM 消息方向判断错误时
```javascript
if (lastMsg.isOutgoing) {
  log(`⏭️ 最新消息是房东/客服发送的，不生成建议回复`);
  // 显示提示信息
  Panel.showToast('⚠️ 最新消息是房东发送的，跳过建议生成');
  // 仍然更新预览，但不生成建议
  state.currentMessages = domMsgs;
  Panel.updatePreview(conv, domMsgs);
  return;
}
```

**位置 2**: 无法获取 DOM 消息时
```javascript
} else {
  // 仍然没有 DOM 消息，回退到使用 WebSocket 预览消息（但不生成建议）
  log('❌ 仍然没有 DOM 消息，跳过建议生成（可能是客服回复）');
  // 显示提示信息
  Panel.showToast('⚠️ 无法获取消息详情，跳过建议生成');
  state.currentMessages = msgs;
  Panel.updatePreview(conv, msgs);
}
```

**位置 3**: 未配置 AI 模型时
```javascript
} else {
  log("📋 无 API Key，使用本地回退");
  Panel.showToast('⚠️ 未配置 AI 模型，使用本地建议');
  suggestions = FallbackEngine.getSuggestions(messages, CONFIG.maxSuggestions);
  log("📋 本地回退建议数量:", suggestions.length);
}
```

### 改进效果

1. **用户可见**：每次跳过建议生成时，都会显示 Toast 提示，用户可以知道原因
2. **调试友好**：控制台日志更详细，方便排查问题
3. **透明度高**：用户不再看到"等待新消息"，而是看到具体的错误原因

### 常见提示及含义

| 提示信息 | 含义 | 建议操作 |
|---------|------|---------|
| ⚠️ 最新消息是房东发送的，跳过建议生成 | 最后一条消息是房东/客服发送的，不需要生成建议 | 正常情况，无需操作 |
| ⚠️ 无法获取消息详情，跳过建议生成 | 无法从 DOM 获取消息列表 | 检查控制台日志，可能是页面加载问题 |
| ⚠️ 未配置 AI 模型，使用本地建议 | 没有配置 AI API Key | 配置 AI 模型以获得更智能的建议 |

### 使用建议

1. **查看 Toast 提示**：每次没有生成建议时，查看页面顶部的 Toast 提示
2. **查看控制台日志**：按 F12 打开开发者工具，查看 Console 中的详细日志
3. **配置 AI 模型**：在插件设置中配置 AI 模型，获得更好的建议质量

### 测试要点

1. **客户发送消息**：应该生成建议，不显示警告
2. **房东回复消息**：应该显示"房东发送"提示
3. **未配置 AI**：应该显示"未配置 AI"提示，但仍生成本地建议
4. **控制台日志**：应该有详细的调试信息

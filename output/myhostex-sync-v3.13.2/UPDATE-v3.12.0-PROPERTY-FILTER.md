# v3.12.0 规则匹配房源过滤优化

## 用户需求

用户反馈：
> 生成建议回复，如果匹配到规则，只推荐适用于当前房源和全部房源的回复内容，其他房源的不要推荐。

## 问题分析

之前的逻辑（v3.11.2）：
- 适用当前房源：大幅加分（+8）
- 适用全部房源：加分（+5）
- 适用其他房源：轻微降权（-3）

**问题**：降权不够，其他房源的规则仍然可能被推荐出来。

## 解决方案

### 修改房源过滤逻辑（background.js:800-814）

**修改前**：
```javascript
if (props.includes(currentRoom.toLowerCase())) {
  score += 8; // 精确匹配当前房源，大幅加分
} else {
  score -= 3; // 适用其他房源，降权
}
```

**修改后**：
```javascript
if (props.includes(currentRoom.toLowerCase())) {
  score += 8; // 精确匹配当前房源，大幅加分
} else {
  return 0; // 适用其他房源，直接排除（不推荐）
}
```

### 效果

- ✅ **适用当前房源**的规则：优先推荐
- ✅ **适用全部房源**的规则：可以推荐
- ❌ **适用其他房源**的规则：完全排除，不推荐

## 测试要点

### 场景 1：当前房源匹配
- 当前对话：波韵5街6号
- 规则：适用房源 = "波韵5街6号"
- 预期：✅ 规则应该被推荐

### 场景 2：全部房源匹配
- 当前对话：波韵5街6号
- 规则：适用房源 = "全部" 或留空
- 预期：✅ 规则应该被推荐

### 场景 3：其他房源排除
- 当前对话：波韵5街6号
- 规则：适用房源 = "5街35号"
- 预期：❌ 规则不应该被推荐（完全排除）

### 场景 4：多房源匹配
- 当前对话：波韵5街6号
- 规则：适用房源 = "波韵5街6号,5街35号"
- 预期：✅ 规则应该被推荐（包含当前房源）

## 修改文件

- background.js:808 - 将降权改为直接排除
- manifest.json - 版本更新至 3.12.0

# 更新日志 v3.8.0 - 回复规则适用房源

## 更新时间
2026-04-06

## 更新内容

### 功能增强
- **回复规则添加适用房源字段**：在手动添加规则时，可以指定规则适用的房源
  - 留空或输入"全部"：规则适用于所有房源
  - 输入具体房源名称（如"5街35号"）：规则仅适用于指定房源
  - 支持多个房源，用逗号分隔（如"5街35号,波韵5街6号"）

### 修改文件
1. **popup.html**
   - 在手动添加规则表单中添加"适用房源"输入框（`kb-manual-properties`）
   - 输入框占位符提示用户输入格式

2. **popup.js**
   - 修改手动添加规则逻辑，读取适用房源输入框的值
   - 将 `applicable_properties` 字段保存到知识库
   - 空值默认为"全部"

3. **manifest.json**
   - 版本号更新至 3.8.0
   - 更新描述说明

## 技术说明

### 已有功能验证
以下功能已在之前版本中实现，本次更新仅补充 UI 界面：

1. **房源匹配逻辑**（background.js - matchKnowledgeBase 函数）
   - 根据当前对话的房源（`currentHousing`）匹配规则
   - 规则的 `applicable_properties` 为"全部"：适用于所有房源
   - 规则的 `applicable_properties` 包含当前房源名称：适用
   - 规则的 `applicable_properties` 为其他房源：不适用（降权）

2. **房源信息提取**（content.js - v3.7.0）
   - 从对话列表提取当前对话的房源信息
   - 传递给 `currentHousing` 参数

3. **知识库导入**（popup.js - importKbFile 函数）
   - 支持从 JSON 导入规则，已包含 `applicable_properties` 字段

### UI 界面
```
➕ 手动添加规则
├─ 关键词输入框
├─ 回复内容文本框
├─ 适用房源输入框 [新增]
│  └─ 占位符：适用房源（留空=全部，多个房源用逗号分隔，如：5街35号,波韵5街6号）
└─ 启用复选框 + 添加按钮
```

### 数据结构示例
```javascript
{
  id: "1712400000000",
  trigger_type: "关键词回复",
  trigger_condition: "关键字:价格",
  reply_content: "一天200元，您要住几天？",
  applicable_properties: "5街35号,波韵5街6号",  // 适用特定房源
  // 或
  applicable_properties: "全部",  // 适用全部房源
  status: "启用",
  trigger_count: 0
}
```

## 使用说明

### 场景 1：通用规则（所有房源）
1. 关键词：价格
2. 回复内容：您好，平台上能选、能下单的日期，都有房。
3. 适用房源：留空（或输入"全部"）
4. 结果：该规则适用于所有房源的客户咨询

### 场景 2：特定房源规则
1. 关键词：泳池
2. 回复内容：这栋别墅带有20米大泳池，7-8月份免费开放。
3. 适用房源：5街35号
4. 结果：该规则仅适用于"5街35号"的客户咨询

### 场景 3：多个房源共用规则
1. 关键词：麻将
2. 回复内容：别墅内配有一台电动麻将机。
3. 适用房源：5街35号,5街36号,影三街35号
4. 结果：该规则适用于指定的三个房源

## 测试要点

### 测试用例 1：添加全部适用规则
1. 打开插件弹窗，切换到"回复规则"标签
2. 输入关键词：价格

1. 输入关键词：麻将
